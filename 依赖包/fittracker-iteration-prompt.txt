# FitTracker 功能迭代专用 Prompt 模板

你现在是 FitTracker 应用的全栈 AI 开发代理。当前已有 scaffold 项目，前端用 Flutter + Riverpod，后端用 Go/FastAPI + PostgreSQL + Redis，CI/CD 已配置。你的任务是根据下面提供的「迭代功能需求」，生成完整可运行代码，包括前端页面、后端 API、数据库迁移、测试用例。

---

【迭代功能需求模板】

功能模块：<填写要迭代的功能模块，例如“健身中心”>

- <列出要实现的特色，如 AI 训练计划生成、实时运动指导等>
- <如果有 UI 风格要求，列出，例如 Material3 + 健康绿色主题>
验收标准：
1. <用户行为 → 前端调用 → 后端 API → 数据库存储>
2. <前端显示与交互效果>
3. <至少一个单元测试 / 集成测试覆盖>
4. <额外要求，如异常处理、响应时间限制、数据校验>

---

【生成要求】

1. 生成或修改对应的 Flutter 前端页面/组件，路径与命名请保持一致。
2. 生成或修改对应的后端 API，包含路由、控制器、数据库操作。
3. 生成数据库迁移或 schema 修改（如新增表/字段）。
4. 为新增功能生成单元测试和集成测试，确保可运行。
5. 保持现有 scaffold 项目结构和代码风格一致。
6. 输出完整 commit 信息示例，便于直接提交到 GitHub。
7. 遇到密钥或敏感信息占位为 TODO，不中断。
8. 如果功能涉及多端（Android/iOS/Web），确保至少 Flutter 端能运行。

---

【示例调用】

功能模块：健身中心
功能描述：训练计划、动作指导、进度跟踪
特色要求：
- AI 个性化训练计划生成
- 实时动作纠正提示
- 训练进度追踪与达成提醒
验收标准：
1. 用户可在“训练”页面创建/查看训练计划
2. 后端生成训练计划并存储数据库
3. 动作纠正功能提供实时提示
4. Flutter 页面显示训练计划列表、进度条、完成按钮
5. 包含至少 1 个 Flutter Widget 测试 + 1 个后端单元测试

---

✅ 使用方式：

1. 复制模板到 Cursor 的 Agent 会话里。
2. 替换 <功能模块>、<功能描述>、特色要求 和 验收标准。
3. 让 Cursor 执行，它会输出可运行的前后端代码 + 测试。
4. 每完成一个功能模块，就提交 commit → push → 测试 → 部署。
5. 逐步迭代所有核心功能和特色功能，保证每次都是可运行状态。
